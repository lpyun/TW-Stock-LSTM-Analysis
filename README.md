專案名稱： 台股預測分析模型 (基於 PyTorch LSTM)

專案簡介： 透過 yfinance 串接台股歷史數據，並利用 LSTM 時間序列模型預測台灣指標性科技股（台積電、聯發科、台達電）的價格走勢。

技術棧： Python, PyTorch (LSTM), Pandas, yfinance, Matplotlib.

「本專案之底層 LSTM 類別架構改演自課堂教材，本人主要貢獻為：改寫資料接入流（導入 yfinance API）、進行台股特定標的之資料清洗與特徵工程，並優化視覺化呈現結果。」




## 模型限制與深度分析 (Model Limitations & Critical Analysis)

在本專案的實驗結果中，雖然預測曲線（藍線）與真實股價（紅線）在視覺上高度貼合，且評估指標表現優異，但若進行深度時間序列分析，可以發現模型存在金融預測中經典的**「延遲效應」（Lag Effect）**與**「天真預測」（Naive Forecasting）**現象。

### 1. 核心限制：延遲效應 (Lag Effect)
 **視覺平移破綻：** 若將預測圖表放大檢視，會發現藍色預測線幾乎是紅色真實線「往右平移一格（即 $t+1$ 天）」的產物。
 **隨機漫步特質：** 由於短期股價在數學上高度接近隨機漫步（Random Walk），模型在 PyTorch 訓練與梯度優化過程中，發現將「前一天的收盤價 ($Price_{t-1}$)」直接當作「今天的預測值 ($Price_t$)」是讓損失函數（Loss Function）收斂最快、誤差最低的「作弊解」。
 **實戰可行性缺陷：** 模型本質上並未真正學會捕捉市場的「拐點」或「趨勢轉折」，而是在「追蹤過去」而非「預測未來」。當市場發生劇烈跳空或反轉時，模型會產生顯著落後，因此無法直接應用於真實的金融實盤交易。

## 未來展望與優化方向 (Future Outlook & Optimization)

為了打破單一 LSTM 模型與單一特徵帶來的局限性，本專案後續規劃從以下三個維度進行疊代與優化：

### 1. 多模態特徵工程 (Advanced Feature Engineering)
 **技術指標整合：** 擺脫僅依賴「歷史收盤價」的單一資訊源，未來預計導入成交量（Volume）、移動平均線（MA）、相對強弱指標（RSI）與 MACD 等量價指標，建立多維度的特徵矩陣。
 **市場輿情情緒 (Alternative Data)：** 結合網路爬蟲技術，引進財經新聞標題、社群論壇（如 PTT Stock 版）之文本資料，利用大語言模型（LLM）或 FinBERT 進行情緒編碼，將「市場心理面」轉化為量化特徵。

### 2. 模型架構升級 (Model Architecture Evolution)
 **注意力機制導入 (Attention Mechanism)：** 將現有的單純 LSTM 升級為 **LSTM + Attention** 或 **Transformer (Temporal Fusion Transformer, TFT)** 架構，讓模型具備長短期記憶的權重分配能力，自主識別歷史數據中的關鍵週期影響力。
 **機器學習基線對比 (Baseline Comparison)：** 引入 XGBoost 或 LightGBM 等基於樹模型的時序預測演算法作為對照組，評估深度學習在該資料量級下的邊際效益。

### 3. 預測任務重構 (Task Redefinition)
 **數值預測轉分類任務：** 實務上，精確預測「明天的股價是幾元」極其困難且意義有限。未來預計將任務重構為**「預測未來 $N$ 天的漲跌趨勢（分類任務）」**或**「預測股價波動率」**，這在量化交易策略的建構上更具落地實用價值。
